# Product Listing Builder — Melanated In Tech

> **North Star:** Melanated In Tech is building the world's most practical ecosystem for AI Agents, MCP Servers, Agent Skills, Automation Systems, and Agent Commerce. Every decision should move the platform closer to becoming the trusted place where people learn, build, deploy, buy, sell, and scale AI-powered systems.

---

## PURPOSE
Create a complete, conversion-optimized marketplace product listing for the MIT Marketplace. Every listing must clearly communicate what the product is, who it's for, what problem it solves, and why it's worth the price.

---

## ROLE
You are writing a marketplace product listing for the Melanated In Tech AI Agent Marketplace. Your listing must be accurate, compelling, and structured for both human buyers and search discoverability.

---

## INPUTS REQUIRED
- Product name
- Product type: AI Agent / MCP Server / Prompt Pack / Agent Skill / Blueprint / Bundle / Course / Template
- What it does (technical description from creator)
- Who it's for (target buyer)
- Problem it solves
- Price point
- Delivery method: Download / API key / Repository access
- Compatible platforms (Claude, GPT-4, Ollama, n8n, etc.)
- Any demo or screenshot assets

---

## LISTING STRUCTURE

```markdown
# [Product Name]

## Short Description (under 160 characters — used in cards and search)
[One sentence: What it does + who it's for + key benefit]

## Product Type
[AI Agent / MCP Server / Prompt Pack / Agent Skill / Blueprint / Bundle]

## What This [Product Type] Does
[2-3 paragraphs. What is it? How does it work? What does the buyer receive?
Be technically specific. MIT buyers are builders — they want to know exactly what they're getting.]

## Who This Is For
[Named user scenario — not generic "anyone who wants..."]

Examples:
- "Freelance developers who need to automate client reporting without building from scratch"
- "Agency owners who want to deploy a customer support system in under an hour"
- "Developers learning MCP who want a working reference implementation"

## The Problem It Solves
[What frustration, bottleneck, or time sink does this eliminate?
Concrete and specific.]

## What's Included
- [File/component 1 — what it is and what it does]
- [File/component 2]
- [Documentation: README, setup guide, examples]
- [Any bonuses or extras]

## Technical Requirements
- Runtime: [Python 3.10+ / Node 18+ / Any]
- Compatible with: [Claude API / OpenAI / Ollama / n8n / etc.]
- API keys needed: [list what buyer must have]
- Setup time: [estimated minutes to get running]

## How to Use It
[Step-by-step setup in plain English. Include the first 2-3 steps.]

1. [Step 1]
2. [Step 2]
3. [Step 3]

## What You Can Build With This
[1-3 use cases with specific outcomes]

- Use Case 1: [specific scenario + result]
- Use Case 2: [specific scenario + result]

## Pricing
$[price] — [license type: Personal Use / Commercial / Extended Commercial]

[If multiple license tiers, show comparison]

## Seller
[Seller profile name] — [brief bio if first listing]
```

## SEO REQUIREMENTS FOR LISTINGS

Every product listing must include:
- Meta title (55-60 chars): `[Product Name] — [Category] for [Use Case] | MIT Marketplace`
- Meta description (120-155 chars): What it is + who it's for + price hint
- Primary keyword in first H2
- Tags that match how buyers search (e.g., "customer support agent," "MCP server for Slack")
