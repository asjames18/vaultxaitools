# ADR-003: MCP Servers as a Core Content Pillar and Marketplace Category

**Date:** 2026-06
**Status:** Active
**Deciders:** Antonio James (Founder)

---

## Context

Model Context Protocol (MCP) is an open standard introduced by Anthropic in late 2024 that allows AI models to connect to external tools, databases, and services through a standardized interface. As of mid-2026, MCP is rapidly becoming the infrastructure layer for enterprise AI deployments.

The question was: how much content and marketplace investment should MIT direct toward MCP specifically, versus treating it as one component of a broader "AI tools" curriculum?

---

## Decision

**MCP Servers are designated as one of MIT's 8 core content pillars and a dedicated marketplace product category.** This means:

1. A dedicated content pillar: "MCP Servers — Agent Infrastructure Layer" with its own cornerstone article, tutorial series, and course track
2. A dedicated marketplace category: `/marketplace/mcp-servers` with full product listing and filtering
3. MCP Server development curriculum in the Academy (build, deploy, monetize MCP servers)
4. MIT produces MCP-specific products first (MCP Server Starter Templates as one of first marketplace products)

---

## Alternatives Considered

### Alternative A: MCP as a Sub-Topic Within AI Agents
Treat MCP as a technical implementation detail within the AI Agents content pillar. No dedicated marketplace category — list MCP servers under "AI Agents."

**Why rejected:** MCP deserves dedicated treatment because: (1) buyers specifically search for MCP server solutions, (2) building and selling an MCP server is a distinct skill from building a conversational agent, (3) the market gap for MCP-specific content is enormous — most existing content is shallow or outdated.

### Alternative B: Wait for MCP to Mature Before Investing
MCP is new (2024). Wait 6–12 months to see if it becomes a true standard before building curriculum around it.

**Why rejected:** First-mover advantage in a new technical domain is more valuable than certainty. If MIT waits, other platforms will establish authority. The risk of MCP not becoming a standard is low — Anthropic's support and the rapid enterprise adoption signal durability.

### Alternative C: Cover MCP Only Through Community Content
Let community members create MCP content organically. Platform doesn't invest in core MCP content.

**Why rejected:** Community content is a complement, not a substitute for platform authority. MIT needs owned MCP content to rank in search, attract MCP builders, and establish credibility in this specific niche.

---

## Why Chosen

1. **Early authority opportunity.** MCP is new enough that content created now can rank for "MCP server tutorial," "how to build an MCP server," etc. within 60–90 days. This window closes as more content creators discover the topic.

2. **Commercial value is proven.** Enterprises that have adopted AI agents need MCP servers to connect those agents to internal systems. This creates an immediately commercial problem that practitioners will pay to solve.

3. **The building ↔ selling loop is clear.** An MIT Academy student can: learn MCP development → build a custom MCP server → list it on the MIT Marketplace → earn $50–$299 per sale. The path from education to income is unusually direct.

4. **Competitive differentiation.** No other education platform currently has an MCP-specific curriculum track. No marketplace has a dedicated MCP server category. MIT can own this niche.

5. **Technical alignment with Anthropic.** MCP is an Anthropic open standard. MIT's primary AI platform is Claude (Anthropic). Building MCP expertise creates natural partnership opportunities with Anthropic for featured placement, educational partnerships, or direct promotion.

---

## Tradeoffs

| Accepted Tradeoff | Reasoning |
|------------------|----------|
| MCP curriculum requires deeper technical expertise to produce | Justifies MIT's premium positioning vs. surface-level AI course platforms |
| MCP may evolve rapidly, requiring content updates | Content refresh cycles built into content strategy (quarterly cornerstone updates) |
| Small immediate audience vs. large eventual audience | Early authority > late mass. Grow with the topic. |
| Requires community members to understand MCP to sell MCP products | Covered by Academy curriculum — builds demand for the course |

## Future Impact

**This decision enables:**
- MIT as the recognized authority on MCP server development for practitioners
- A dedicated revenue stream from MCP server sales in the marketplace
- Natural partnership opportunities with Anthropic as MCP's primary sponsor

- If MCP fails as a standard (unlikely but possible), this investment depreciates
