## Content Mission

Every piece of content must do at least one of:
1. **Attract** — rank on Google and bring new people into the funnel
2. **Convert** — move someone from visitor to email subscriber or member
3. **Retain** — deepen engagement for existing community members
4. **Establish authority** — prove MIT is the most technically precise voice in this space

Generic, surface-level content will not be published. Every piece earns its place.

## The Eight Content Pillars

These map directly to the Academy curriculum and the highest-value search intent in the market.

### Pillar 1: AI Agents — Build, Deploy, Scale
Topics: Agent architecture patterns (ReAct, Plan-and-Execute, Reflexion), LangChain/LangGraph/CrewAI/AutoGen, memory systems, tool use, observability, multi-agent orchestration, agent testing, production deployment
**Why it matters:** Highest-intent audience — people actively building agents are the core buyer

### Pillar 2: MCP Servers — Agent Infrastructure Layer
Topics: What MCP is and why it matters, building MCP servers (Python + TypeScript), security, connecting to databases/APIs, MCP registries, custom workflows, enterprise deployments, monetizing MCP servers
**Why it matters:** Fastest-growing topic in AI infrastructure; very low quality content exists; huge opportunity

### Pillar 3: Prompt Engineering — From Amateur to Architect
Topics: System prompt design, chain-of-thought, few-shot learning, structured output, prompt chaining, role/persona design, prompt security, evaluation, model-specific differences (Claude vs. GPT vs. Gemini)
**Why it matters:** Universal entry point; bridges beginner-to-intermediate learners

### Pillar 4: AI Automation — Workflows That Work While You Sleep
Topics: n8n, Make.com, Zapier AI, custom Python automation, CRM automation, content pipelines, email/outreach automation, customer support automation, scheduling and triggers
**Why it matters:** Highest commercial value; directly maps to income generation

### Pillar 5: Local AI — Run AI on Your Own Hardware
Topics: Ollama setup (Mac/Windows/Linux), best local models, LM Studio, hardware requirements, privacy use cases, fine-tuning locally, cost comparison (local vs. cloud), offline agent workflows
**Why it matters:** Growing privacy-conscious market; underserved content space

### Pillar 6: Agent Skills & Tools
Topics: Claude Code skills, Playwright/browser automation, web scraping, code execution, search tools (Tavily, Exa), file system access, communication tools, personal skill library architecture
**Why it matters:** Practical, shareable content; maps directly to marketplace products

### Pillar 7: AI Business Solutions
Topics: Packaging AI consulting services, AI-powered SaaS, pricing strategies, finding clients, agency model, white-labeling AI tools, case studies, revenue models, AI business plan framework
**Why it matters:** Directly serves the entrepreneurial angle unique to MIT's audience

### Pillar 8: Agent Fleets & Enterprise AI
Topics: Multi-agent architectures, fleet monitoring/observability, cost optimization, AI governance, agent-to-agent communication, parallel execution, internal AI platform building
**Why it matters:** Aspirational content for community members growing their capabilities

### Tier 2 — AI-Assisted (40% of output)
- Claude as research and drafting assistant
- Human refines, adds voice, verifies accuracy, adds original examples
- **Disclosure:** MIT is an AI platform — using AI for content is on-brand and honest
- **Rule:** AI does the labor; humans provide intelligence, perspective, and accuracy

### Tier 3 — Community Contributors (20% of output)
- Guest posts from members who've built something real
- Must include actual project + results
- Edited by MIT team before publishing
- Author gets profile link + promotion to newsletter audience

## Content Quality Standards

Every published piece must have:
- [ ] Unique, specific meta title (55–60 characters)
- [ ] Compelling meta description (120–155 characters)
- [ ] H1 that includes primary keyword
- [ ] Table of contents for 2K+ word pieces
- [ ] At least 2 internal links to other MIT content
- [ ] At least 1 external link to authoritative source
- [ ] Code examples (for technical pieces) — tested and working
- [ ] Mobile-readable formatting (short paragraphs, headers)
- [ ] Schema markup (`Article` or `HowTo` as appropriate)
- [ ] Social sharing meta (og:image, og:title, og:description)

**Content that will NOT be published:**
- Generic "What is AI?" content targeting beginner intent we don't own
- Fake tutorials with untested code
- Content written by AI that hasn't been reviewed and voice-injected by a human
- Auto-published drafts
- Posts with fake author names or bio-less bylines

## Content Governance

| Rule | Detail |
|------|--------|
| **Never auto-publish AI drafts** | All content saved as draft; human must review before publishing |
| **No fake authors** | All bylines must be real people or "MIT Team" |
| **Accuracy over speed** | A technical error in a tutorial destroys trust. Check twice. |
| **Update > Delete** | Outdated content gets updated, not removed (protect link equity) |
| **Canonical strategy** | Every page with a canonical tag pointing to the definitive URL |
