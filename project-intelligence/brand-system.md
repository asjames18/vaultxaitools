## Visual Identity

### Color Palette

| Role | Color | Hex | Usage |
|------|-------|-----|-------|
| Background Primary | Near-Black | `#0A0A0A` | Page backgrounds, cards |
| Background Secondary | Dark Gray | `#111111` | Elevated surfaces |
| Background Tertiary | Charcoal | `#1A1A1A` | Input fields, borders |
| Accent Primary | Neon Green | `#00FF41` | CTAs, highlights, active states |
| Accent Secondary | Bright Green | `#00CC33` | Hover states, secondary CTAs |
| Text Primary | White | `#FFFFFF` | Headlines, body text |
| Text Secondary | Light Gray | `#A0A0A0` | Subtext, meta information |
| Text Tertiary | Mid Gray | `#666666` | Disabled, placeholders |
| Danger | Red | `#FF4444` | Errors, destructive actions |
| Warning | Amber | `#FFB800` | Warnings, pending states |
| Success | Green | `#00FF41` | Confirmations (same as accent) |

**Key Rule:** The neon green `#00FF41` on near-black `#0A0A0A` is the visual signature. It should never appear on light backgrounds. It is always a signal of action, importance, or energy.

### Typography

| Role | Font | Weight | Size |
|------|------|--------|------|
| Display Headlines | Geist (or Inter) | 700–900 | 48px–96px |
| Section Headlines | Geist | 600–700 | 24px–40px |
| Body Text | Geist | 400 | 16px–18px |
| Code / Technical | Geist Mono | 400 | 14px–16px |
| Labels / UI | Geist | 500 | 12px–14px |

```js

  'mit-black': '#0A0A0A',

  'mit-gray': '#A0A0A0',

```

### Visual Style Principles

1. **Dark-first** — All designs start on dark backgrounds. Never invert to light mode for core branding.
2. **Minimalist** — Space is a design element. Do not crowd. Let content breathe.
3. **Technical precision** — Clean lines, no decorative flourishes that distract from content
4. **Neon accents intentionally** — Green only appears on CTAs, active states, highlights. Overuse kills the effect.
5. **Bold typography** — Large, confident type signals authority and directness

## Brand Voice

### Core Voice Attributes

| Attribute | What It Means | What It Never Means |
|-----------|--------------|---------------------|
| **Bold** | Direct, confident, declarative | Aggressive, exclusionary, arrogant |
| **Grounded** | Honest, practical, real-world | Academic, theoretical, detached |
| **Precise** | Technically accurate, specific | Jargon-heavy, inaccessible |
| **Forward** | Future-oriented, possibility-focused | Hype, overpromising, FOMO |
| **Unapologetic** | Certain of our mission, uncompromising on standards | Defensive, combative, self-righteous |
| **Empowering** | Builds capability, not dependency | Performative, empty affirmation |

### Tone Adjustments by Context

| Context | Tone Shift |
|---------|-----------|
| Tutorial / Technical content | More precise, more structured, let quality speak |
| Community posts | Warmer, more peer-to-peer, still direct |
| Marketing / Headlines | Most bold, most punchy, visceral |
| Error messages | Clear and helpful, never condescending |
| Success messages | Celebratory but brief — respect the user's time |
| Email newsletters | Founder voice — personal, direct, like a trusted peer |

## Brand Archetype Deep Dive

**The Mentor Who Made It and Came Back**

This archetype is central to every content decision. Ask: would the mentor who actually built and deployed AI agents say it this way?

| The mentor IS | The mentor is NOT |
|--------------|------------------|
| Technically rigorous | A cheerleader |
| Culturally fluent | Performatively inclusive |
| No-nonsense | Gatekeeping |
| Genuinely rooting for success | Giving empty praise |
| High standards | Unnecessarily harsh |
| Precise and specific | Vague and motivational |

## What We Never Do

- We never use corporate-speak or DEI buzzwords
- We never use filler phrases: "Unlock your potential," "Transform your career," "Join the revolution"
- We never write passive voice when active works
- We never overpromise outcomes (income, success, time-to-results)
- We never publish AI content without human review and voice injection
- We never use fake social proof (testimonials, followers, results) we haven't earned
- We never simplify AI concepts to the point of inaccuracy to appear accessible

---

## Proof Points (Earned, Not Manufactured)

These only go into marketing once verified:
- Purpose-built curriculum on AI Agents, MCP Servers, and automation
- Marketplace where community lists and sells real AI tools
- Founder with hands-on technical AI building and deployment experience
- Community architecture for peer learning, not passive consumption
- Content built for income-generating systems, not engagement metrics
